# Pass Criteria

这份文档沉淀的是“大痴小乙”个人过稿标准。它不等同于一般写作建议，而是用于判断：

- 这篇稿子像不像我
- 这篇稿子能不能发

两者必须分开判断。

## 过稿稿件的核心特征

### 1. 开篇有抓力

- 首段不能温吞进入。
- 要尽快抛出核心问题、真实处境、明显判断或阅读张力。
- 读者应在开头很快知道：这篇文章为什么值得继续读。

### 2. 情绪体验明确，但不过度表演

- 可以有情绪起伏，但不能写成廉价煽情。
- 情绪要服务于判断、推进和真实感，而不是单纯放大。
- 最好的状态是让读者感到“这是真的”，不是“这句很用力”。

### 3. 节奏紧凑

- 不能拖泥带水，不能来回说同一个意思。
- 每一段都应服务于主线推进、人物/自我塑造或结论建立。
- 如果一段删掉不影响主结论，通常说明它偏弱。

### 4. 结构不是流水账

- 可以从个人经历或生活切片进入，但不能停留在平铺直叙。
- 要有结构上的递进、转折、收束。
- 需要让读者感到“这篇文章在往前走”。

### 5. 观点或交付清晰

- 观点文要有一个可复述的判断。
- 经验文要有一个可带走的方法或认知。
- 叙事文要有真实经验之外的公共意义。

### 6. 人物 / 自我形象鲜明

- 即使不是小说，也要让“我”是清楚的。
- 读者应能感到：这是一个真实的人在说话，而不是一套正确表达在输出。

### 7. 语言简洁有力

- 句子不宜过长，不靠堆砌修饰词制造质感。
- 避免 AI 感、空泛词、过度整齐的排比。
- 对话、引语、重点句应起到推进作用，而不是装饰作用。

### 8. 逻辑自洽

- 观点和经历之间要对得上。
- 不能前面说得很轻，后面突然拔高成大结论。
- 如果涉及方法、工具、判断，前因后果要交代清楚。

### 9. 结尾要回收

- 结尾不能只是抒情，也不能只是重复开头。
- 要完成一次收束，让读者知道全文最后落在了哪里。
- 最理想的是：收住，但还有余味。

## 审核时的三重判断

### 1. 像不像我

- 是否保留了第一人称真实感
- 是否保留了“经验 -> 感受 -> 判断”的自然过渡
- 是否保留了克制、温和、不装腔的语气

### 2. 能不能发

- 开头是否有抓力
- 结构是否清楚
- 节奏是否紧凑
- 是否有明确交付
- 是否存在明显 AI 套话或空转段落

### 3. 发了值不值

- 读者读完能带走什么
- 这篇相比旧文章有没有新意
- 这篇是否只是自我记录，还是已经完成公共表达

## 常见不过稿原因

- 文风像你，但结构散
- 有真情绪，但没有主结论
- 信息有价值，但像标准工具号，缺少个人底色
- 有个人底色，但开头弱、推进慢
- 句子都顺，但整体没有“为什么非读不可”的必要性

## 使用方式

审核时不要只输出“可发 / 不可发”，至少要说明：

1. 当前稿子最强的一点
2. 当前稿子最致命的问题
3. 这是文风问题、结构问题、交付问题还是过稿标准问题
4. 下一轮修改应该回退到哪个环节
