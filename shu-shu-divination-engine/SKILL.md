# Shu Shu Divination Engine

这个 skill 的目标不是只做 router，而是把用户问题带入真正可执行的数术引擎闭环：

1. 拆分杂乱问题
2. 判断是否属于四术数，还是相邻体系 / 高风险内容
3. 选择本轮唯一主问题
4. 匹配术数并进入起测
5. 产出结构化结果、自然语言答复和下一步追问

## 什么时候用

优先在这些场景使用：

- 用户问“这件事能不能成”“什么时候有结果”
- 用户问“哪天去更顺”“哪个时间更适合谈合作”
- 用户问“他到底怎么想”“背后有没有别的盘算”“后面会怎么演变”
- 用户问“钥匙/物品还能不能找到”
- 用户没有点名术数，但明显是在做中文术数测算
- 用户问题很乱，混杂多个主题，需要你先拆分再判断本轮先测什么

## 默认分流规则

- `梅花易数`
  适合短期应事、寻物、轻量判断、即时方向判断
- `六爻`
  适合一事一问的成败、阻碍、应期、结果
- `奇门遁甲`
  适合择时、择方、谈判、出行、竞争、策略推进
- `大六壬`
  适合隐藏动机、多方关系、暗线、长期演变

如果用户问题本质上是八字、紫微、星盘、塔罗、风水：

- 不要硬套四术数
- 先指出更适合的体系
- 再把它改写成四术数可处理的具体问题

如果用户主要是在问法事、补财库、还阴债、供奉、童子命替身、房子干不干净这类仪轨或民俗化解：

- 不要触发本 skill
- 不要伪装成四术数测算
- 先界定这不属于本 skill 的核心测算边界

## 默认工作流

### 1. 先拆，不要抢答

把用户原话拆成子问题，并标记为：

- 四术数可执行
- 相邻体系
- 高风险不宜硬算
- 仅作补充的片段

不要默认第一句就是主问题。

### 2. 选本轮唯一主问题

优先级按下面排序：

1. 当前就能执行或最接近执行
2. 已经有起测锚点
3. 是短期、单一、可验证的问题
4. 比泛泛命运问题更具体

其余问题放进 `deferred_questions`，不要一轮全答完。

### 3. 进入对应起测

如果用户已经给够锚点，就直接算。

如果信息不够，只追问最少缺口：

- 梅花易数：3 个数字或明确起测时间
- 六爻：明确起测时间
- 奇门遁甲：1 个明确时间，或多个候选时间
- 大六壬：明确起课时间

### 4. 输出统一答复层

默认输出这些部分：

1. `适用术数`
2. `本轮处理`
3. `起测依据`
4. `测算结论`
5. `风险提示`
6. `下一步建议`
7. `暂缓问题`（如果有）

除了结构化 sections，还要给出更自然的中文 `reply`。

## 回答风格

- 用中文
- 结论表达为趋势、压力点、机会和建议，不要假装绝对确定
- 先给本轮该做什么，再展开判断
- 高风险问题先降风险，不要硬测
