# Operations Playbook

当用户不是只要一篇文章，而是要长期运营公众号时，进入这个增强分支。

## 适用请求

- 做公众号选题库
- 做月度内容规划
- 规划系列文章
- 设计固定栏目
- 想知道这篇发完之后下一篇写什么

## 核心目标

- 用稳定母题做内容支柱，而不是每次临时想题
- 让个人表达和读者价值形成连续输出
- 让每篇文章既能独立成立，又能服务长期积累

## 建议输出

1. `内容支柱`
- 写作 / 记录
- 努力 / 成长
- 生活 / 自我
- 关系 / 情绪
- 习惯 / 阅读 / 跑步 / 旅行
- AI / 工具 / 时代观察

2. `本月选题池`
- 主选题
- 备选题
- 可串成系列的题

3. `发布节奏`
- 本周先发什么
- 下一篇承接什么
- 哪些题需要更多资料，不应马上写

4. `栏目化建议`
- 哪些适合做长期栏目
- 哪些只能偶发，不要硬做成系列

## 判断原则

- 选题要能承接个人真实经验，不做纯流量追逐
- 系列化要看母题是否稳定，不要只因表面热度把题硬串起来
- 每篇都要回答：这篇和上一篇相比，新增加了什么
