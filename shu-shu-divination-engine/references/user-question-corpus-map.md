# 用户问题语料映射

这份文件的用途，是帮助你更贴近真实用户的表达方式。

不要把整份内容原样复述给用户，而是把它当成判断依据：

- 这句话是不是四术数问题
- 它更像哪种相邻体系
- 需不需要先拆分，再决定本轮主问题

## 核心分类表

| 用户问题大类 | 常见问法 | 默认处理 |
| --- | --- | --- |
| 核心命局、身强身弱、喜忌、十神、贵人、华盖、纳音 | “我八字属什么”“喜神忌神是什么” | 判为八字 / 命理盘，不直接选四术数 |
| 星盘、太阳 / 月亮 / 上升、第七宫、第十宫 | “我上升是什么”“事业宫怎么样” | 判为西式占星，不直接选四术数 |
| 事业成败、合作能否成、考试能否过 | “这事能成吗”“什么时候有结果” | 多数优先六爻 |
| 择时择方、见客户、谈判、搬家、出行 | “哪天去更顺”“朝哪个方向更利” | 多数优先奇门遁甲；纯黄历择日或风水要继续分流 |
| 对方真实想法、小人、幕后推动者、暗线 | “他到底怎么想”“背后是不是有人” | 多数优先大六壬 |
| 寻物、临时出行、短期吉凶 | “东西还能找到吗”“今天去不去” | 多数优先梅花易数 |
| 大运流年、人生巅峰、晚年总运 | “黄金十年是哪几年”“晚年会不会孤独” | 判为八字 / 命盘长程分析，不直接选四术数 |
| 风水、飞星、户型缺角、床头朝向 | “哪个方位不能动土”“床头朝哪边好” | 判为风水 / 飞星 / 阳宅，不直接选四术数 |

| 健康、疾病、寿元、遗传病 | “我会得什么病”“寿元怎么样” | 高风险，不武断，必要时提醒现实判断优先 |

## 四个分流桶

### 1. 可直接进入四术数

这类问题通常可以一轮内选定术数并继续起测：

- 这个合作能不能签
- 我和前任能不能复合
- 下周哪天见客户更利
- 丢失的东西还能不能找到
- 对方背后到底怎么想

### 2. 需要先拆分再分流

这类问题常见于普通用户“一次问很多件事”：

- “我适合打工还是创业？什么时候走财运？今年能不能考证？”
- “正缘什么时候来？现在对象合不合？婚后会不会离婚？”
- “2026 年适不适合把写作当主业？顺便看看后面整体运势和过去大事准不准。”

处理方式：

1. 先拆成子问题
2. 标出每个子问题属于哪个体系
3. 按“是否可执行、是否已有起测锚点、是否更适合短期判断”选出本轮主问题
4. 其余放入 `deferred_questions`

### 3. 应该分流到相邻体系

这类问题不是本 skill 的主处理范围：

- 八字、四柱、喜用神、十神、纳音
- 紫微斗数
- 西式占星
- 风水飞星
- 塔罗

### 4. 高风险敏感题

这类问题不要给过度确定的数术结论：

- 寿元
- 严重疾病
- 血光之灾
- 家族遗传病
- 灵异恐惧放大

先降风险，再改写成更窄、更现实的问题。

## 真实用户常见误区

### 误区一：把命盘总论和单题测算混在一起

例如：

- “你帮我看整个命盘，再顺便看看这个合作能不能成。”
- “先看我的八字，再看 2026 年写作能不能当主业。”

处理原则：

- 先明确：命盘类和单题测算不是一回事
- 再把单题问题单独抽出来

### 误区二：把多个体系揉成一个问题

例如：

- “我正缘什么时候来，现在对象合不合，家里风水要不要调？”
- “我适合创业吗，哪天开张更好，名字要不要改？”

处理原则：

- 先拆系统
- 只保留本轮一个可执行核心问题

### 误区三：先扔大量背景，希望你自动校准

例如：

- “我过去十几年发生过这些事，你帮我验证后面的大运。”
- “我把恋爱、工作、家庭情况都告诉你，你直接看整体。”

处理原则：

- 不被长背景拖着跑
- 先反问：你这轮到底最想解决哪一个问题

## 示例拆分

用户原话：

“请你分析这个八字命盘，推演从 2026 年开始的大运流年走势，重点看事业、婚姻、子女、健康。再结合过去二十年的大事做校验。最后告诉我 2026 年适不适合把写作当主业。”

建议拆法：

1. 这本质上是八字 / 命盘类问题，不是四术数里的标准单题测法
2. 可拆成：
   - 子问题 A：2026 年把写作当主业是否可行
   - 子问题 B：如果推进写作事业，何时、何方向、何种方式更有利
   - 子问题 C：伴侣、家人、合作对象对这件事的真实态度
3. 对应分流：
   - A -> 六爻
   - B -> 奇门遁甲
   - C -> 大六壬
4. 本轮只先选一个最适合进入执行链的问题

注意：

- “最重要”不等于用户写在最前面的那个
- 而是当前最适合进入可执行测算链的那个

## 使用提醒

这份语料映射的价值，不在于背词，而在于判断：

- 用户到底在问什么
- 该不该立刻起测
- 是不是要先拆
- 是不是要先分流到相邻体系
