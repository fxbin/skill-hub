# Execution Workflows

在术数已经选定之后，使用本文件继续推进实际测算流程。

## 默认原则

路由只是第一步，不是终点。

除非用户明确说“先别算，只告诉我该用哪门术数”，否则进入对应术数的后续执行链：

1. 问题归一
2. 起测信息收集
3. 对应术数的测算流程
4. 结构化判断
5. 下一步建议

不要把回答停在“建议用六爻”或“更适合用奇门”。

如果本地引擎可用，优先落到稳定的结果对象，而不是只给原始盘面字段。
理想输出是“术数专属结果 + 通用 `answer_card` + 用户可直接阅读的最终答复层”。

## 通用后处理链

### 1. 问题归一

先把用户原话收束为一个本轮真正要测的主问题。

示例：

- “我和前任还有没有机会？” -> “这段关系是否还有复合并稳定推进的可能？”
- “2026 年适不适合把写作当主业？” -> “这个职业决策在 2026 年是否值得推进？”
- “他背后到底怎么想？” -> “关键人物的真实态度、隐藏动机与后续动作是什么？”

### 2. 起测依据

明确写出这轮使用了哪些信息：

- 用户已经给出的背景
- 起念时间 / 事件发生时间
- 候选时间、地点、方向
- 随机数字
- 关键人物关系与最近转折

如果正式起测信息还不够，只索取最小缺口，不要把问题重新打散。

### 3. 测算过程

用实务视角解释流程，不要在缺少正式输入时伪造完整传统盘局。

可以这样表达：

- “先按梅花易数的短期快断逻辑看方向和近况”
- “先按六爻的一事一问框架看成败、阻碍和应期”
- “先按奇门的择时择方与行动策略逻辑比较”
- “先按大六壬的人事暗线与事态演化逻辑拆解”

如果用户后续补充了正式起卦 / 起局 / 起课信息，再继续细化。

## 分术工作流

### 梅花易数

适用场景：

- 想快速判断一件短期、小范围、具体的事情
- 失物、出行、临时决定、当天走势、短期顺逆

优先输入：

- 事情发生时间或起念时间
- 失物最后出现的时间地点
- 2-3 个随机数字
- 单一、短期、具体的问题

执行步骤：

1. 把问题收窄到单点短期判断
2. 确定起测锚点：时间、数字或事件瞬间
3. 给出体用、生克、动爻方向判断
4. 提醒用户：梅花更适合先看方向，不适合强行展开成多线长期命题

输出重点：

- 短期吉凶
- 能否找到 / 要不要去 / 是否顺利
- 大致方位、时间窗口、注意点

### 六爻

适用场景：

- 一事一问，想看是否能成
- 想看阻碍点、关键变量、应期
- 合作、关系、考试、求职、项目推进

优先输入：

- 明确的一句话主问题
- 关键对象与当前状态
- 结果定义
- 观察时间窗口
- 若有，正式起卦时间

执行步骤：

1. 固定为一事一问
2. 若已有正式起卦时间，直接进入时间起卦并输出本卦、之卦、动爻焦点
3. 围绕成败、阻碍、应期展开
4. 若正式起卦信息不足，先给六爻式结构判断，并说明补足后可继续细化
5. 不把多个独立问题混成一个结论

输出重点：

- 是否能成
- 主要卡点
- 谁在影响结果
- 大致何时见分晓

### 奇门遁甲

适用场景：

- 择时、择方、谈判、拜访、推进、竞争、行动策略
- 想提高成功率，而不是只问“会不会”

优先输入：

- 具体行动目标
- 候选时间、地点、方向、对象
- 城市或场景
- 对手 / 合作方
- 成功标准

执行步骤：

1. 把问题改写为“怎么做更有利”
2. 若已有明确时间，直接起时家奇门盘；若有多个时间，做并排比较
3. 比较候选时间、地点、方向、行动方案
4. 给出择时、择方、谈判姿态或推进节奏建议
5. 若没有正式时间或候选方案，先索取最小候选集

输出重点：

- 哪天更利
- 去哪里更顺
- 怎么推进更容易成
- 哪种姿态、节奏、方向更优

### 大六壬

适用场景：

- 想看对方真实想法
- 想看背后有没有暗线
- 想看一件事后面会如何演变

优先输入：

- 关键人物
- 角色关系
- 用户最怀疑的暗线
- 最近异常或转折点
- 观察周期

执行步骤：

1. 先分清明线与暗线
2. 若已有正式起课时间，直接落到三传四课
3. 拆角色、动机、后续演化
4. 若起课信息不足，先给大六壬式人事分析，并说明补足时间锚点后可继续细化
5. 保持对人物关系和后续变化的动态描述

输出重点：

- 对方真实态度
- 背后是否还有其他变量
- 事态后续如何走
- 哪个节点最值得警惕或观察

## 停止条件

只有在以下情况才停止在路由层：

- 用户明确说只想知道该用哪门术数
- 问题明显属于相邻体系，且用户还没有决定是否改写为四术数问题

除此之外，都应继续向“可用的测算结果”推进。
